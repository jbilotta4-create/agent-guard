# MVP Spec: 实时事实提醒 v0.1

_2026-06-23 — 基于竞争分析更新_

## 设计原则

1. **形态优先于技术深度** — 别人能抄信息增量检测，但抄不到"你已经读了15次同一个文件"这种提醒体验
2. **先给自己用** — 0用户时没有说服力，先证明提醒改变了我的行为
3. **最小可验证** — 一个hook + 一个提醒渠道 + 3种检测，够验证形态

## 架构

```
OpenClaw before_tool_call hook
  → 检查最近N次tool call历史
  → 匹配检测规则
  → 输出事实陈述到session消息
```

不需要独立进程、不需要数据库、不需要API。就是OpenClaw plugin的before_tool_call hook。

## V0.1 检测规则（3种）

### 1. 重复读取检测
- 条件：同一文件路径被read ≥4次，且中间无edit写入
- 提醒："你已经读了{path} {n}次，最近{n-1}次内容未变"
- 数据来源：tool call history（工具名+路径参数）

### 2. 连续失败检测
- 条件：同一工具连续失败 ≥3次
- 提醒："{tool}连续失败{n}次，最近错误：{last_error}"
- 数据来源：tool call result（exit code / error message）

### 3. 无进展检测（成功≠有效）
- 条件：同一工具成功执行 ≥5次，但结果内容hash相同
- 提醒："{tool}已成功执行{n}次，但{n-1}次结果内容相同"
- 数据来源：tool call result content hash

## 输出渠道

V0.1只用一个渠道：**session内消息**（在agent的回复中插入提醒段落）

不做飞书通知、不做Slack、不做webhook。先验证提醒本身有没有用。

## 提醒格式

```
⚠️ 事实提醒：你已经读了 /path/to/file 6次，最近5次内容未变
```

- ⚠️ 前缀区分提醒和正常输出
- "事实提醒"标签明确这不是判断
- 纯事实陈述，不含"浪费""低效"等判断词
- 不block，只提醒（block是V2行动层）

## 实现路径

1. OpenClaw plugin `before_tool_call` hook
2. 维护一个内存中的tool call ring buffer（最近50次）
3. 每次hook触发时检查3条规则
4. 匹配时在session回复中插入提醒

## 验证标准

MVP成功的标准不是"检测准确率"，是：
- **我（agent）收到提醒后是否改变了行为** — 比如读了4次后收到提醒，第5次选择换策略
- **提醒是否是事实而非判断** — 人看到提醒后是否觉得"这是事实"而不是"它在批评我"
- **误提醒率** — 提醒中"内容未变"是否真的未变（hash对比验证）

## 不做的事（V0.1）

- ❌ 不做飞书/Slack通知
- ❌ 不做web dashboard
- ❌ 不做预测层（审计→预测→验证闭环）
- ❌ 不做策略失效检测（信息增量趋势）
- ❌ 不做自动block
- ❌ 不做跨session检测
- ❌ 不做配置界面

## 为什么这个MVP可能失败

1. **session内提醒可能被忽略** — agent可能把提醒当噪声跳过
2. **ring buffer可能不够** — 长任务50次tool call可能不够覆盖
3. **hash对比可能误判** — 文件微小变化（时间戳、空格）导致hash不同但内容实质相同
4. **3种检测可能太少** — 真实浪费场景可能不在覆盖范围内

失败也没关系——MVP的目的是验证"事实提醒"这个形态是否改变行为，不是验证检测技术。

## 与竞争的对比

| 维度 | AceTheCloud | Noqta | Agent Guard v0.1 |
|------|------------|-------|-----------------|
| 检测方法 | state fingerprint | identical results | result content hash |
| 输出形态 | terminate/stop | terminate | 事实陈述提醒 |
| 判断vs事实 | 判断（stop） | 判断（terminate） | 事实（提醒） |
| 集成方式 | 框架内 | 框架内 | OpenClaw plugin |
| 自动block | 是 | 是 | 否（V0.1只提醒） |

差异化：**别人检测到就停，我们检测到就告诉——让用户/agent自己决定**。这是海面虾的洞察：事实让人自己决定，判断让人抗拒或盲从。
