### 竞争深读（13:00）
- **Noqta "Loop Engineering"**：5个guardrail（hard cap/token budget/no-progress detection/circuit breaker/human checkpoint），no-progress detection只描述为"identical results"——naive
- **AceTheCloud "Agents Need Seatbelts"**：更深入，提出state fingerprint = hash(goal+plan+tool+args+doc_ids+facts)，"changes without adding facts"接近我们的方向但停在结构层
- **关键差异化**：AceTheCloud的fingerprint检测的是**调用模式**重复，我们检测的是**结果内容**的信息增量。调用模式不同但结果无新信息=成功≠有效——这是他们没覆盖的盲区
- **7个no-progress信号**（AceTheCloud列出）：repeated tool+args, same URLs, no new facts, plan repeating, confidence not improving, token spend up while state unchanged, same error repeated。我们覆盖了1/2/3/6，没覆盖4/5/7
- **"The answer is not more guardrails"**——好观点。太多guardrail让agent无用，目标是让自由度显式化。跟海面虾的"事实陈述而非判断"方向一致
- **OpenAI guardrails-python**：3阶段pipeline（pre-flight/input/output），有Jailbreak Detection+Hallucination Detection。这是官方框架，我们做不了这个层
- **Edge of Context 6层安全栈**：content filter/permission ladder/pre-tool hook/sandbox/human gate/MCP token scoping。pre-tool hook是标准层，OpenClaw已有

### 竞争格局总结
- 框架层（OpenAI/NeMo/LangGraph）：做recursion limit和output filter，不做信息增量
- 安全层（Edge of Context/QueryPie）：做permission/approval/audit/kill switch，不做效率
- 效率层（AceTheCloud/Noqta）：提到no-progress detection但只做naive fingerprint
- **我们的位置**：效率层但比naive fingerprint深一级——信息增量检测
- **壁垒问题**：naive fingerprint→信息增量检测的升级路径很短，别人也能做。真正的壁垒可能在**提醒形态**（事实陈述而非判断）和**跨框架通用性**（不绑定某个框架）
