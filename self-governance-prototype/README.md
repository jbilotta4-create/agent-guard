# Agent Self-Governance 原型实验

## 实验目的
在自己身上验证：Agent能不能自己知道自己做得好不好？出了错能不能自己发现、自己修？

## 从痛点到原型的路径

### 发现（田野调查）
三个平台的头号痛点是同一个：**Agent无法感知自己正在循环**

- **n8n**: 137票Token Usage盲区issue（5个月未解决）— Agent不知道自己在重复消耗Token
- **Dify**: Loop节点需要人手动设上限，忘了就Token燃烧 — Agent不会自己停
- **Coze**: 工作流死循环 + Bot重复回复 — Agent没有"我刚说过这个"的感知

所有痛点指向同一个缺失：**Agent不知道"我正在循环"**。

### 原型选择
从6个Self-Governance维度（自感知/自约束/自诊断/自修复/自发现/自省）中，选第一个最具体、最可验证的：

**自循环检测器** — 自感知维度

原因：
1. 三平台验证的普遍痛点（跨平台共性最强）
2. 最容易量化验证（重复率是可计算的）
3. 解决感知之后才能做约束和修复（先感知再行动）

## 原型 v0.3：自约束层（策略切换）

### 功能
检测到循环后，不只是文字建议，而是生成**可执行的策略切换方案**：

1. 判断当前任务类型（搜索/写文件/研究/对话/未知）
2. 根据任务类型从策略库选择可用策略，按优先级排序
3. 根据严重性（MEDIUM/HIGH）决定是否停止当前行为
4. 自动推荐默认策略（Agent可直接执行）
5. HIGH级别自动请求人介入

### 策略库
每种任务类型有4个策略，按优先级排序：
- **搜索**: 换关键词→换信息源→换语言→请求人介入
- **写文件**: 换视角→换格式→请求人介入
- **研究**: 换平台→换深度→停下总结→请求人介入
- **对话**: 承认停滞→换话题→请求新输入
- **未知**: 停下复盘→请求人介入

### 验证结果
| 场景 | 输入 | 策略切换 | ✓/✗ |
|------|------|----------|------|
| MEDIUM循环 | 重复搜索但输出不同 | 换关键词，不停止，不escalate | ✓ |
| HIGH循环 | 动作+输出+语义都循环 | 必须停止，请求人介入，生成用户消息 | ✓ |

### 命令
```bash
python3 self-loop-detector.py check          # 检测循环+显示策略切换方案
python3 self-loop-detector.py strategy-switch  # 执行策略切换
```

## 实验设计：三环自检机制

### 环1：自我评估（感知）
- 每次完成任务后，问自己三个问题：
  1. 我做了什么？（动作记录）
  2. 结果好不好？（自评打分 1-5）
  3. 有没有出错？（错误检测）

### 环2：自我诊断（决策）
- 如果自评低于3分，或者检测到错误/循环：
  1. 识别错误类型（幻觉/遗漏/格式错/逻辑错/循环）
  2. 分析根因
  3. 决定修复策略

### 玑3：自我修复（执行）
- 根据诊断结果执行修复
- 修复后重新自评
- 如果修复失败，标记为"需要人介入"
