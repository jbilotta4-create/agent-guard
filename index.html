How It Works

1

Hook into every tool call

Agent Guard runs as a plugin hook inside your agent runtime. Every tool call passes through it before execution.

2

Detect loops in real-time

Same tool + same parameters repeated? Same tool called excessively? Consecutive errors? Detected and logged instantly.

3

Block before the call executes

When a loop is confirmed, the tool call is blocked before it reaches the API. No wasted tokens. No runaway bills.

4

Self-governance against over-blocking

If Agent Guard itself blocks too many consecutive calls, it auto-downgrades to detection-only mode. Governance tools need governance too.
